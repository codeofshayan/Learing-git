Configure the Git
    - Once the Git is installed we need to configure the Git.
    - Configuring simply means we are informing the git that in which account we are making changes.
    - i.e. Whenever we use the git to make changes on the github which email should be associated with this which name should be tracked and so on.

Code to configure
    > git config --global user.name "usernameOfYourChoice"
    > git config --global user.email "email@google.com"

To check using git config what settup we have created.
    > git config --list
    - This will show the credentials of our git including our username and email.

Basic Commands
    > git clone
        used for cloning the repository on our local machine.
        clone the repo using this command in terminal and use cd repo name to get inside it. 
    > git status
        displays the status of the code
    > git add <-file_name->
        adds new or changed file in your working directory to Git staging area
        we can add individual files either by name.
        for multiple we simply say : git add .
    > git add .
        adds all the new changes to the staging area.
    > git commit -m "some message"
        It is a record of change.
    > git push origin main
        push uploads the local repo content to the remote repo
        git push : pushing the repo
        origin   : (from where we have cloned)we are calling our remote repo as origin
        main     : it is our branch
    > git pull origin main
        pulls the latest remote repo content to your local repo
        i.e. if someone else had made any changes and you just write this command.
        all the new files and modification will be updated to your local
    > git restore <-file_name->
        restore the repo to the previous version, from where you have started change for this file.
    > git restore .
        restores all the file.

Get inside the desired repo

        - .git/ is the folder that git uses to helps you in tracking the git repo 
        - after init you can use this to test wether the repo is initialized or not
    > git init
        - It will give similar prompt like this (Initialized empty Git repository in C:/Users/Shayan/Desktop/Learning git/Local-repo/.git/)
        - Your local repo has has initialized the remote repo.
        - i.e remote repo has been created
    > we will go to github UI
        - create new repo, without readme file
        - once the repo is created you can see all the basic steps are given to you.
    > git remote add origin <-link->
        - we are asking to add a remote whose name will be origin and assigning URL as your origin.
        - git remote add origin https://github.com/codeofshayan/localrepo.git
    > git remote -v
        - to verify the remote repo
    > git branch
        - Tells you on which branch you are.
        - * star denotes which branch you are on.
        - master is the default branch
    > git branch -M main
        - git branch -M <-new main-> 
        - This renames the name of the existing branch.
    > git push origin main
        - push the local repo to the remote repo
    > git push -u origin main
        - (-u) : means set up streams
        - we are using the "-u" as flag
        - (-u) is a universal flag that says "I'll be pushing to the origin main always"
        - what happens from next time you can use only "git push" to push your file to the origin main
        - we use this if we have a very long application and every time we are commiting then this helps us to reduce that load.
    > git push
        - can only be used after you have flagged with the universal flag.
    > we can create : README.md file
        - in the VS code locally
    > git pull origin main
        - If some modification is made on the remote side.
        - By doing this you are updating you local repo by fetching latest pull from the origin itself.




=================================   GitHub WorkFlow


GitHub > clone > changes > add > commit > push 

1. Create repo using the UI : Create a repo on GitHub
2. git clone <-url->        : Clone the repo to you local
3. changes or new files/folder : Make the desired changes and modification
4. git add .                : add the file to the repo
6. git commit -m "<-decription->" : commit the changes you made
7. git push origin main     : Push the changed to origin






========================   BRANCH Commands
When there are multiple developer associated with 1 repo.
They create there seperate branch for seperate feature.

    > git branch
        - to check the current branch
    > git branch -M <-new-branch-name->
        - to rename the existing branch name into the desired name
    > git checkout <-branch-name->
        - to navigate
        - to move from one branch to another
    > git checkout -b <-new-branch-name->
        - to create new branch
    > git branch -d <-branch-name->
        - ti delete the existing branch
    > git restore --staged <file>
        - use "git restore --staged <file>..." to unstage
        - file goes to staging once we write > git add .




way-1
    > git diff <-branch name->
        - To compare the commit, branches, files and more
        - branch name will be the main, As you will have to compare it with the original/previous version of repo.
        - git diff main
        - you can exit with q, :q
    > git merge <-branch name->
        - this allows you to merge the branch.
        - git merge main : will merge the feature1 branch content to the main


way-2
    pull request (PR)

- Once you merged the feature branch to main branch using the PR.
- You will still see that the content is not available in the main branch it is only available in the feature branch
- To see the latest changes "git pull origin main" ***.
